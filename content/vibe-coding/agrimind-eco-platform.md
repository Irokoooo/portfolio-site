## 需求觉察（Motivation）
这份调研报告的终点，原本只是一份躺在电脑夹里的 Word 文档。但在密云区金叵罗村实地走访时，一位村民的无心之问彻底击中了我：“你们年轻人都说 AI 赋能，但这 AI 到底怎么能赋能我们农民啊？”这种宏大科技叙事与乡村真实泥土感之间的巨大撕裂，让我意识到，仅仅指出“信息不对称”“品牌缺失”是不够的。我想要打造一个真实的“助农一体化”网页场景，砍掉冗长的中间商环节，让农产品的流转可视化，用真实的代码去回答那位村民的问题。

## 痛点与挑战（Pain Points）
当真正开始用 Vibe Coding 把田野调查转化为网页时，理想与现实的骨感瞬间暴露无遗：

1. 非结构化农业数据的“量化困境”。AI 极其擅长处理格式化的 JSON 数据，但真实的农业太“重”了。泥土湿度、西红柿酸甜度、村民口口相传的种植经验，这些充满在地质感的信息，根本无法直接变成代码参数。
2. 脱离硬件支持的“空中楼阁”。我在网页规划里设计了溯源和实时监控功能，却立刻撞上物理世界的墙：没有田间物联网传感器与硬件配套，前端展示的数据都会沦为空转数据。
3. 步子迈太大的“渲染崩溃”。作为新手，我曾试图让 AI 一次生成包含密云全景地图和 3D 农场交互的完整系统，结果代码逻辑层层嵌套，浏览器持续报错，AI 生成结果也陷入循环冲突。

## 落地解法（Implementation）
面对被现实与 Bug 双重痛击的局面，我重新调整了人机协作策略与自己的知识边界：

1. 化整为零的 Prompt 拆解。放弃“一键生成 3D 农场”的幻想，将庞大需求拆为最小可行产品（MVP）：先完成基础农产品图文信息流，再逐步加入轻量交互动效。并通过阅读开源项目代码，反向指导 AI 微调 CSS 与 JS 细节。
2. 跨学科补课，打通软硬件逻辑。既然代码卡在硬件环节，我去计算机学院补物联网与硬件基础，理解传感器数据如何经 API 进入后端。即使不亲自焊电路板，也要先打通底层数据链路，才能让 AI 生成的接口具备落地可能。

## 迭代心得（Iteration Notes）
从踩在密云泥土里做访谈，到最终捧回全国三等奖，这个项目重塑了我对“科技助农”与“Vibe Coding”的认知。我意识到，光靠键盘和几句华丽 Prompt 远远不够。AI 确实降低了写代码门槛，但也更容易让人陷入功能堆砌，最终留下大量看起来完整却没有业务价值的模块。真正的科技向善不是泛泛炫技，而是双脚沾泥、基于真实田野痛点做针对性开发。Vibe Coding 的意义不仅在于“普通人也能写代码”，更在于让我们在看见现实痛点后，有能力做出哪怕很小但真实有效的改变。
