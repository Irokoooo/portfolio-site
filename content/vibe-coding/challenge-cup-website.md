## 需求觉察（Motivation）
为了参与 2024 年的“挑战杯”全国大学生创业计划竞赛，我们团队将助农出海的目标锁定了陕西洛川。洛川苹果品质极佳，但当地农户直接对接海外市场时面临着巨大的语言和沟通壁垒。这个项目的底层逻辑非常明确：利用大语言模型 API 和数字人技术，搭建一个跨国商务销售平台。通过 AI 实现精准翻译与产品推荐，解决海外客户获取信息的障碍，从而切实帮助洛川果农拓宽海外销售渠道。

## 痛点与挑战（Pain Points）
项目从基础的网页展示升级为引入 AI 数字人交互，主要面临两个技术逻辑卡点：

1. AI 知识库的精准度要求：如果只接入通用的大模型 API，AI 只能给出空泛的回答。我们需要将整理好的洛川苹果特定数据（口感偏甜/脆、品种差异、储藏方式等）无缝嵌入到对话上下文中，才能让 AI 给出有效的精准推荐。
2. 数字人主动式交互的实现：要将团队设计的“苹小柚”形象套图转化为网页上能互动的实体。这不仅需要前端代码能够根据对话状态切换静态图以模拟口型和表情，还需要系统能监测用户的浏览行为，让数字人能够在合适的时机主动发起推销对话，这涉及复杂的前端状态管理。

## 落地解法（Implementation）
1. 设定结构化 Prompt 与本地知识库：我放弃了通用对话模式，转而利用结构化 Prompt 限制大模型的回答边界。将洛川苹果的具体卖点和参数作为前置知识喂给 AI Agent，确保海外客户在询问时，API 能调取这些业务数据，给出像专业销售一样的准确推荐。
2. 状态流驱动的 2.5D 数字人：针对数字人实现，我通过 AI 辅助编写了前端脚本，将设计的“苹小柚”图片素材建立为一个状态库（如空闲、打招呼、介绍产品）。通过监听 AI API 的文本返回结果，触发前端 JS 脚本进行图片切换，低成本实现了网页端的可视化交互。
3. 埋点触发主动对话逻辑：为了让数字人具备主动销售能力，我让 AI 补充了前端用户行为监测代码。当系统监测到用户在某个苹果品种页面停留超过设定时间时，会自动触发数字人发送特定开场白（如主动介绍该品种的脆甜度），进而引导用户进入销售对话流。

## 迭代心得（Iteration Notes）
最终这个项目拿到了挑战杯全国二等奖，这也是我个人第一个真正实现商业化落地的项目。这给了我极大的鼓励，它不仅验证了 Vibe Coding 结合具体商业场景的可行性，更让我借此机会，成功将自己前期自学和积累的前端交互设计知识（如状态管理、行为监听、动效触发）实打实地落实到了实际产品中，完成了从概念到可用产品的闭环。
