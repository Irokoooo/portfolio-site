## 需求觉察（Motivation）
受家庭背景以及个人对于法律的深厚兴趣影响，我一直关注法律实务领域。在学习国际经济与贸易专业的《合同法》时，我观察到一个明确商业痛点：企业在出海或跨国贸易中，面临极高的涉外合同审查成本和合规风险。这个项目的底层逻辑在于，虽然涉外合规高度依赖专业法律知识，但大语言模型具备强大的文本抽取和规则匹配能力，可以将繁琐的初步审查流程自动化。恰逢学姐邀请我参加全国“智法杯”人工智能赋能法学创新大赛，我决定以此为契机，开发一个涉外法务合规 AI 平台，验证这一逻辑的商业可行性，并尝试 agent 的商业实战。

## 痛点与挑战（Pain Points）
作为一个非计软背景的商科生，我在全栈开发中遇到了两个核心卡点：

1. Agent 接入与后端部署障碍。此前我的 Vibe Coding 经验仅限静态网页生成，从未尝试将腾讯云等企业级大模型 Agent 接入自定义系统。面对晦涩的官方部署文档、API 鉴权规则以及 Render 服务器的后端配置，前期经历了漫长技术试错。
2. 前端界面排版简陋。早期单纯依靠 Prompt 生成的网页布局较为平白，缺乏视觉层级，无法满足“专业法务平台”应有的 UI 标准。

## 落地解法（Implementation）
1. Copilot Plan 模式控制框架。我改变了让 AI 一次性输出整套代码的策略，转而使用 VS Code Copilot 的 Plan 模式。先用自然语言敲定平台整体架构和核心功能逻辑（如总体大纲、针对性修改逻辑、变动记录等），确立基础规范后，再逐个组件生成与优化。
2. 以原型设计指导前端代码。为解决前端布局问题，我系统学习网页设计基础和 Figma。通过绘制高保真原型、学习设计专业语言，并配合额外 AI 进行结构化前端 Prompt 编写，从而精确控制 AI 生成组件的尺寸、色彩与排版。
3. 构建轻量级后端处理 API。在接入腾讯云 Agent 时，结合论坛与社媒前辈经验，我意识到 API 密钥安全和会话上下文维持的重要性。我利用 AI 辅助编写了轻量级后端中转服务，通过该后端完成身份鉴权、Session ID 管理，并编写脚本解析 Agent 返回的复杂 JSON 数据，将其精准映射到前端对应展示面板。

## 迭代心得（Iteration Notes）
在独立完成这个具备专属 3D IP 和完整业务流的系统后，我对 Vibe Coding 最大的感悟是：AI 并没有消除写代码的门槛。如果缺乏干预，AI 极易产出冗余、重复内容，最终堆砌成难以维护的“屎山代码”。因此，作为项目主导者，人类核心价值在于清晰的需求表达能力与基础代码逻辑判断力。开发者必须能够精确指出代码中的逻辑断层，并指导 AI 进行针对性优化。
