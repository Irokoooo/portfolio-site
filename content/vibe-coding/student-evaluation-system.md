## 需求觉察（Motivation）
大一下学期期末计算综测和绩点时，我作为班委遇到了一个极其繁琐的管理痛点：同学们的活动证明材料发得非常零散，且学校的综合素质测评评分标准复杂冗长。因为对细则理解不同，每个人自己算出来的分数出入很大，导致班委需要花费大量时间和精力去反复收集材料并人工核查。我的直觉是：综测本质上是一套高度制度化、规则明确的运算逻辑，这种完全规则驱动的工作本就应该交给系统来自动化处理，而不是依赖人工算账。因此，我决定利用 AI 辅助开发一套标准化的“大学生综测系统”。

## 痛点与挑战（Pain Points）
在利用 Vibe Coding 从零搭建这个系统时，我面临的主要技术与逻辑卡点在于：

1. 复杂计分规则的代码化：学校的综测细则不仅包含基础绩点计算，还包括极其繁杂的加分项（如国家级、省级、校级权重不同，且各项加分有严格的上限封顶）。如果一次性把规则丢给 AI，生成的计算逻辑往往会发生冲突或遗漏边界条件，导致算出的总分不准确。
2. 证明材料的结构化流转：系统不仅是一个计算器，还需要具备材料收集与审核功能。这就要求开发一套完整的前后端交互界面，前端让学生分类上传证明截图或文件，后端让班委清晰地进行打回或通过操作。对于非科班学生来说，理顺这种包含数据绑定和状态扭转的业务流难度很大。

## 落地解法（Implementation）
1. 将管理规则转化为算法 Prompt：为了保证计算绝对准确，我放弃了让 AI 直接写个综测系统的粗放提问。我先将学校的《综测细则》人工拆解为清晰的逻辑树和数学条件判断式，然后分模块（如德育模块、智育模块、文体模块）把规则喂给大语言模型。通过这种“投喂清晰数学边界”的 Prompt 策略，让 AI 生成了准确无误的 JavaScript 计分函数。
2. 分步构建双端业务控制台：在搭建材料流转系统时，我规划了“学生端（提交与进度查看）”和“班委端（审批与一键核算）”两条链路。通过指导 AI 逐个编写表单组件和数据看板，成功串联了材料上传、分类存储和状态标记（待审/通过/驳回）的完整工作流，彻底代替了以前混乱的 Excel 表格与微信群收集。

## 迭代心得（Iteration Notes）
这套系统经过后续的 UI 美化和代码优化后，我将其作为成果提交给了学校，目前该系统正在落实并部署到“智能民大”的官方信息系统中。这个从班级痛点一路走到校级部署的项目让我深刻体会到：Vibe Coding 的核心威力，在于打破“业务需求”和“技术实现”的壁垒。当身为班委的你对现实管理痛点有极其深刻的体感时，AI 就是最好的执行工具，它能把你清晰的管理思维和业务逻辑，低成本且精准地转化为一套真正能够解放生产力的制度化系统。
