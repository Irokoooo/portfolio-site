## 一、网站核心定位（Core Positioning）

### 表面功能
一个极简、现代的**个人数字简历与作品集载体**，主要用于生成 QR Code 附加在 PDF 简历上，直接赋能 Erasmus（伊拉斯谟）等"商科+Tech"跨学科硕士项目的申请。

### 深层逻辑
一个**动态生长的"个人数字花园"与"能力档案馆"**。不仅面向 HR 和招生官，更是作者自我梳理、记录跨界产出（商业、技术、语言、设计）的长期个人资产库。

### 目标受众优先级
1. **P0**：欧洲高校招生官（Erasmus Mundus 等项目）
2. **P1**：国内外 HR 与企业招聘方
3. **P2**：作者本人（自我成长记录与能力梳理）

## 二、已确认功能模块

### ✅ 模块 1：首屏引言区（Hero Section）

**目标**：访客在 10-15 秒内建立清晰的人设认知

**必须包含**：
- 精炼的中英双语 Personal Tagline
- 核心优势组合：`国际经贸底色 × AI 工具链落地 × 多语种跨文化`
- 下载 PDF 简历按钮（直接触发文件下载，无跳转）
- 社交媒体链接区（GitHub、LinkedIn、邮件）
- 头像或个人照片（可选，保持极简）

**禁止包含**：
- 大型轮播图、自动播放动画
- 超过 3 行的自我介绍段落
- 复杂的粒子效果或 3D 背景

### ✅ 模块 2：技能生态档案（Skill Ecosystem）

**目标**：将碎片化技能分类归档，提供所见即所得的"工作证明（Proof of Work）"

**4 大技能分类**：

#### 2.1 产品与设计
- Luma Flow UI 草图
- Canva 海报设计作品
- Logo 迭代记录

#### 2.2 数据与分析
- 特高压经济影响研究数据可视化
- CDA 备考逻辑导图
- Python / Stata 分析记录

#### 2.3 语言与写作
- 脱敏后的高质量学术文书
- 多语种（英/法）沟通证明材料
- 精美专业课 PPT 节选

#### 2.4 AI 辅助工程
- Vibe Coding 实践记录（本网站构建过程）
- Prompt 优化心得与案例
- AI 工具链使用记录

**交互规则（渐进式披露 Progressive Disclosure）**：

| 交互层级 | 触发方式 | 展示内容 |
|---------|---------|---------|
| 默认态 | 无 | 技能标签 + 分类图标 |
| 轻交互 | 鼠标 Hover | 缩略图预览 或 一句话数据提示 |
| 重交互 | 点击 | Accordion 展开 或 Modal 弹窗（沉浸式展示 PPT/海报/PDF） |

### ✅ 模块 3：项目画廊（Project Gallery）

**目标**：以 Notion Board View 风格直观展示主打项目

**卡片内容规范**（每张卡片必须包含）：
- 高质量封面图（16:9 或 4:3 比例）
- 项目标题（≤20 字）
- 一句话描述（≤50 字）
- 技能标签（2-4 个，如 `Product Management` `Vibe Coding` `Python`）
- 时间标签

**布局规范**：
- 桌面端：3 列网格
- 平板端：2 列网格
- 移动端：1 列纵向

### ✅ 模块 4：项目详情页（Project Detail）

**目标**：承接卡片点击，展示解决复杂问题的完整逻辑闭环

**固定结构（CRPO 框架）**：
```
Context  → 项目背景：为什么做这个？
Role     → 我的角色：我在其中承担什么？
Process  → 执行过程：如何拆解与执行？（可含图表/截图）
Outcome  → 可量化产出：做到了什么效果？
```

**必须包含**：
- 返回画廊的面包屑导航
- 项目相关技能标签
- GitHub 源码跳转按钮（若有）
- 外部演示链接（若有）
- 阅读时间估计

## 三、UX/UI 设计原则

### 视觉哲学
**Notion 风格极简主义**：
- 大量留白（Whitespace as a design element）
- 白底黑字为主，内容本身即设计
- 低饱和度辅助色，不做彩色堆砌
- 排版本身即美感，无需过多装饰元素

### 信息架构原则
**渐进式披露（Progressive Disclosure）**：
- 首屏只展示最核心的 20% 信息
- 用户主动探索才能看到 80% 的细节
- 避免信息过载，减少认知负担

### 性能优先
- 首屏加载 < 2s（Lighthouse Performance Score > 90）
- 图片全部使用 Next.js `<Image>` 组件自动优化
- 静态页面优先（SSG），减少服务器负载

### 无障碍设计
- 所有图片必须有 `alt` 文本
- 颜色对比度符合 WCAG AA 标准
- 键盘导航完整支持
